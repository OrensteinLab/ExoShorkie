# Distillation

## Synthetic Sequence Generation + Ensemble Prediction

This script generates **AlphaGenome-style synthetic DNA windows** (strand flip + ~4% SNVs + small SVs) from FASTA sequences, then runs an **ensemble of Shorkie/NatShorkie models** to predict RNA-seq coverage for each synthetic window.  
It saves a compressed `.npz` file that can be used for **distillation / student training**.

---

## What the script does

1. Loads sequences from one or more FASTA sources.
2. Builds fixed-length windows of size **16,384 bp** (`WINDOW_BP`).
3. Samples windows and applies augmentations:
   - 50% reverse-complement
   - ~4% base substitutions (SNVs)
   - Poisson(λ=1) structural variants: insertion / deletion / inversion (max 20 bp)
   - Crops/pads back to exactly 16,384 bp
4. Runs an ensemble of models to predict each synthetic sequence.
5. Averages predictions across all models and saves outputs to an `.npz`.

---

## Modes

### 1) Genomic mode (zero-shot NatShorkie)

Use **NatShorkie** models trained on native yeast genome and predict on synthetic windows sampled from **all yeast chromosomes**.

- Model template:

```text
Models/NatShorkie/f{fold}/model_finetune.h5
```

- Requires:
  - `--src1-fasta` (yeast reference FASTA containing chrI–chrXVI records)

### 2) Finetuned mode (exogenous-specific ensemble)

Use models fine-tuned for a specific genome/chromosome (and optionally a second source).

- Model template:

```text
Models/{chrom}/cv{cv}/f{fold}/model_finetune.h5
```

- Requires:
  - `--src1-chrom`, `--src1-fasta`
- Optional:
  - `--src2-chrom`, `--src2-fasta`

In this mode, the ensemble iterates over **5 CV folds** (`cv0..cv4`) × **8 ensemble members** (`f0..f7`) = **40 models**.

## Usage

### A) Genomic mode (NatShorkie, yeast chromosomes)

```bash
python scripts/distill_synthetic.py \
  --genomic \
  --src1-fasta Data/genome/S288c_reference.fa \
  --n-synth 50000 \
  --target-windows 10000
```

### B) Finetuned mode (single source)

```bash
python scripts/distill_synthetic.py \
  --src1-chrom Mpneumo \
  --src1-fasta Data/genome/Mpneumo.fa \
  --n-synth 50000 \
  --target-windows 10000 \
  --ensemble 1 
```

## Arguments

- `--genomic`  
  Use NatShorkie (zero-shot) models and sample windows from **all yeast chromosomes**.

- `--src1-chrom`  
  Source/chromosome name (used for model path in finetuned mode and output naming).

- `--src1-fasta` *(required)*  
  FASTA file for source 1.

- `--src2-chrom`, `--src2-fasta` *(optional)*  
  Optional second source (finetuned mode only).

- `--n-synth` *(default: 50000)*  
  Number of synthetic sequences to generate.

- `--target-windows` *(default: 10000)*  
  Target number of genomic windows to sample from when building the pool.

---

## Output

The script writes:

```text
Results/Distillation/<chrom_for_path>/synthetic_<chrom_for_path>_mean_preds.npz
```

Where `<chrom_for_path>` is:

- `genomic` in `--genomic` mode  
- otherwise the value of `--src1-chrom`

### Contents of the `.npz`

- `train_sequences`  
  Object array of synthetic DNA strings (each length 16,384 bp)

- `train_mean_preds`  
  `float32` array of ensemble-mean predictions for each synthetic sequence  
  Shape is whatever the model outputs per window (e.g., `(n_synth, n_bins)`)

# Train student model

This script trains a lightweight **student Shorkie model** by distilling knowledge from an ensemble of pretrained teacher models.

Instead of using experimental RNA-seq labels, the student is trained directly on the **ensemble-mean predictions** produced by the synthetic generation + ensemble inference pipeline.

---

## What this script does

1. Loads synthetic sequences and ensemble predictions from:

```text
Results/Distillation/<chrom>/synthetic_<chrom>_mean_preds.npz
```

2. Builds a student model consisting of:

- The pretrained Shorkie trunk 
- A new dense distillation head 

3. Trains the student to match teacher outputs using:

- Mean Squared Error loss (MSE)
- Pearson correlation metric
- Spearman correlation metric

4. Saves distilled student weights to:

```text
Distillation/<chrom>/student_<chrom>_distilled.h5
```

---

## Input File

The script expects the `.npz` file generated by the synthetic distillation pipeline.

### Required keys inside the NPZ

- `train_sequences`  
  Synthetic DNA windows (length = 16,384 bp)

- `train_mean_preds`  
  Ensemble-mean teacher predictions per sequence  
  Shape: `(N, 896)` (896 coverage bins)

---

## Model Initialization

The student model is initialized from the pretrained Shorkie trunk:

```text
Models/shorkie/f0/model_best.h5
```

Only the trunk weights are loaded, and a new head is trained.

---

## Usage

Run distillation by specifying the chromosome/genome name:

```bash
python scripts/train_student.py \
  --chrom Mpneumo
```

---

## Hyperparameters (defaults)

| Parameter     | Value      |
|--------------|------------|
| Window size   | 16384 bp   |
| Epochs        | 20         |
| Batch size    | 64         |
| Learning rate | 2e-5       |
| Output bins   | 896        |

---

## Output

After training, distilled weights are written to:

```text
Results/Distillation/<chrom>/student_<chrom>_distilled.h5
```

---

## Metrics Reported

During training, the script reports:

- **MSE loss**
- **Pearson correlation** across bins per sequence
- **Spearman correlation** (Pearson on ranks)

# In-Silico Mutagenesis (ISM) for Distilled ExoShorkie Student Models

This script performs base-resolution **in-silico mutagenesis (ISM)** over an entire genome or chromosome using a distilled student ExoShorkie/Shorkie model.

For every nucleotide position, the script generates all **3 possible single-base substitutions**, runs the model prediction, and measures the effect on total predicted RNA-seq coverage.

The computation is optimized to run almost entirely on the GPU using batched mutation generation.

---

## What this script does

For a given genome FASTA and a distilled student model:

1. Loads the chromosome sequence
2. Splits it into overlapping 16,384 bp windows
3. Predicts baseline RNA-seq coverage for each window
4. Mutates each interior position to all alternate bases
5. Re-predicts coverage for each mutation in large GPU batches
6. Computes the log2 fold-change effect of each mutation
7. Writes a genome-wide ISM score map to disk

---

## Inputs

### Required arguments

| Argument | Description |
|---------|-------------|
| `--model` | Model name (used to locate distilled weights) |
| `--chrom` | Chromosome identifier in the FASTA file |
| `--fasta` | Path to genome FASTA file |
| `--mu` | Mean used for log z-score normalization |
| `--sigma` | Std-dev used for log z-score normalization |

### Optional flag

| Flag | Description |
|------|-------------|
| `--rev` | Run on the reverse-complement strand |

---

## Example Usage

### Forward-strand ISM

```bash
python scripts/ISM_student.py \
  --model Mpneumo \
  --chrom Mpneumo \
  --fasta Data/genome/Mpneumo.fa \
  --mu 2 \
  --sigma 2
```

### Reverse-complement ISM

```bash
python scripts/ISM_student.py \
  --model Mpneumo \
  --chrom Mpneumo \
  --fasta Data/genome/Mpneumo.fa \
  --mu 2 \
  --sigma 2 \
  --rev
```

---

## Model Loading

The script loads a distilled student model from:

```text
Results/Distillation/<model>/student_<model>_distilled.h5
```

---

## Windowing Strategy

The genome is processed in windows of:

- Window size: **16,384 bp**
- Cropped edges: **1,024 bp** on each side
- Interior mutated region: **14,336 bp**
- Stride: **14,336 bp** (non-overlapping interiors)

Mutations are only applied in the interior to avoid boundary artifacts.

---

## Output

### Output file location

ISM scores are saved as a memory-mapped binary file:

```text
Results/ISM/Maps/<chrom>/<model>_model_chr_<chrom>_<fwd|rev>.dat
```

### Output shape

The output is a float32 matrix of shape:

```text
(L, 4)
```

Where `L` is the chromosome length and columns correspond to mutated base channels:

| Column | Base |
|-------|------|
| 0 | A |
| 1 | C |
| 2 | G |
| 3 | T |

Each entry stores the **log2 fold-change** in predicted total coverage when mutating that base.

---

## Mutation Effect Score

For each mutation, the script computes:

```math
\Delta = \log_2 \left( \frac{\text{mut\_sum} + 1}{\text{baseline\_sum} + 1} \right)
```

---

## Performance Notes

- Mutations are processed in large GPU batches (`MUTATION_BATCH_SIZE=512`)
- Mutation generation + prediction happens entirely on GPU via `tf.function(jit_compile=True)`
- Results are streamed to disk using NumPy `memmap` to avoid RAM overload

---

## Requirements

- NVIDIA GPU strongly recommended
- Distilled student models already trained
